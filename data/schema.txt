# Legend:

/Case/etc/etc - xpath for xml element
+{n,m} suffix - there are n to m of these elements per case
() suffix     - contains names of important xml attributes
-> ...        - element contains child elements that were not mapped

####################
# Mapping to SQLite:

- Main case data in 'Cases' table
- Forms mapped to individual tables
- Elements have same name as xpath in SQLite, but replace '/' with '_'
- For XML attributes, add '_' and attribute name as suffix
- Special characters removed (eliminate '+', change '-' to '_')
- All tables have 'CaseId' to use as a key
- XML prefix is shortened in SQLite tables according to table below

NAME             XML PREFIX                                     PREFIX IN SQL
Cases            /Case                                          Case
Events           /Case/CaseForm/Events                          Event
Vehicles         /Case/CaseForm/Vehicles/VehicleSum             Vehicle
Persons          /Case/CaseForm/Persons/Person                  Person
EMS              /Case/EMSForm/EMSvehicle                       EMS
GeneralVehicle   /Case/GeneralVehicleForms/GeneralVehicleForm   GeneralVehicle
Occupants        /Case/OccupantForms/OccupantForm               Occupant
Safety           /Case/SafetyForms/SafetyForm                   Safety
VehicleExterior  /Case/VehicleExteriorForms/VehicleExteriorForm VehicleExterior
VehicleInterior  /Case/VehicleInteriorForms/VehicleInteriorForm VehicleInterior
